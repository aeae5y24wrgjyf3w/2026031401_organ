# 2026031401_organ
波形間で共通する倍音の重なりを補正するオルガン

## 概要
周波数比が簡単な整数比の音程にある音同士を重ね合わせると、それぞれがもつ倍音のうち、同じ周波数をもつもの同士の間で干渉が発生する。本リポジトリは、のこぎり波や矩形波を干渉の悪影響が存在しないように、整数比の周波数比で重ねる方法を提案する。

## 言語と開発環境
- 言語 : C
- OS : Windows11
- コンパイラ : gcc version 8.1.0

## 使用方法と仕様
### コンパイル
2026031403_static_saw.c、2026031404_dynamic_saw.c、2026031405_dynamic_saw_square.cのいずれかひとつと2026031402_io.hと2026031406_smf2wav.cをいっしょにコンパイルすること。

### プログラムの基本仕様
WAV化したいSMFのファイル名を引数にして実行すると、カレントディレクトリにWAVファイル(a.wav)が生成される。\
対応するMIDIイベントはノートオン、ノートオフ、速度変更のみ。ほかのイベントは無視される。\
波形は正規化されてWAVファイルに書き込まれる。サンプルレート44100Hz、ビット深度16ビット、モノラル。\
実行にかかる時間は曲の長さに比例する。

### 音域、調律、振幅、波形
#### 音域
ノートナンバー24以上の音を鳴らすことができる。
#### 調律
調律は実装により異なる。（実装上の制約ではなく恣意的に決定されている。）
#### 振幅
波形の振幅はMIDIベロシティの平方根に比例する。
#### 波形
2026031403_static_saw.c,2026031404_dynamic_saw.cではMIDIチャンネルに関係なくすべての音の波形はのこぎり波になる。\
2026031405_daynamic_saw_square.cではのこぎり波に加え、矩形波もを使うことができる。MIDIチャンネルが偶数の音はのこぎり波、奇数の音は矩形波で鳴る。\
矩形波は同じベロシティで鳴らしたのこぎり波より弱い。矩形波の音のベロシティをのこぎり波の音のベロシティの4/3倍にすると、同じくらいの強さに聞こえる。

## 改変へのヒント
各音の各実装において調律はstatic FREQ tuning[12]でMIDIノートナンバー24から35までの周波数として設定されている。ほかのオクターブでの周波数はこれの2の冪倍になる。任意の整数値が可能なので、tuning[12]の値を変えることでほかの調律で鳴らすことができる。あるいは根本から周波数割り当て規則を変えたりすることもできる。\
リアルタイム用途で使う場合など、ホストプログラムが振幅を正規化しない場合は、絶対振幅がホストプログラムの受け入れられる範囲を超えないように、どこかの段階で振幅を調整しなければならい。
